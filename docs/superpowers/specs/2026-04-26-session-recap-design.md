# Session Recap & QR Takeaway — Design Spec

## Overview

After a voice conversation on the AskGordie kiosk, a QR code appears on screen alongside the Gordie badge. Users scan it to view their conversation on their phone — a branded recap page with chat history, sources, and a CTA to sign up for CanadaGPT. Sessions are stored locally in SQLite and synced to Supabase for multi-kiosk visibility.

## Data Model

### SQLite (local on each Pi)

```sql
CREATE TABLE sessions (
    id TEXT PRIMARY KEY,           -- UUID
    device_id TEXT NOT NULL,       -- e.g. "gordie-001"
    started_at TEXT NOT NULL,      -- ISO 8601
    ended_at TEXT,                 -- NULL until conversation ends
    topic_count INTEGER DEFAULT 0,
    synced INTEGER DEFAULT 0,      -- 0=false, 1=true
    expires_at TEXT NOT NULL,      -- started_at + 24h default
    scanned INTEGER DEFAULT 0      -- 0=false, 1=true (QR was scanned)
);

CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    role TEXT NOT NULL,             -- "user" or "gordie"
    content TEXT NOT NULL,
    sources TEXT DEFAULT '[]',      -- JSON array of {title, url}
    created_at TEXT NOT NULL        -- ISO 8601
);

CREATE INDEX idx_messages_session ON messages(session_id);
CREATE INDEX idx_sessions_synced ON sessions(synced) WHERE synced = 0;
CREATE INDEX idx_sessions_expires ON sessions(expires_at);
```

### Supabase (cloud, same schema)

Tables: `kiosk_sessions` and `kiosk_messages` with identical columns plus Supabase-managed `created_at` timestamps. The `device_id` column enables multi-kiosk queries.

## Conversation Flow

1. **Wake word detected** → New session created in SQLite with UUID + device_id. `expires_at` set to now + 24 hours.
2. **Each Q&A pair** → Two messages saved (role=user, role=gordie). Sources extracted from Gordie's response and stored as JSON.
3. **Conversation ends** (user says goodbye, walks away via presence timeout, or idle timeout) → `ended_at` set, `topic_count` calculated from distinct user messages.
4. **QR code appears** alongside the badge in the bottom-right corner. CTA: "Scan to save your conversation."
5. **New user arrives** (presence detected or wake word) → QR stays visible for 10 more seconds, then fades out. New session starts immediately and independently.
6. **User scans QR** → `scanned` set to true, `expires_at` extended to 30 days.

## QR Code

- Generated by Flask route: `GET /qr/session/<uuid>`
- Points to: `http://gordie.local:8080/s/<uuid>`
- Displayed via SocketIO event `session_ended` with the UUID
- Frontend shows QR with fade-in animation in bottom-right corner of voice view
- Size: approximately 6-7 inches on screen (scannable from 4-7 feet)
- Dismissal: fades out 10 seconds after a new conversation begins, or 30 seconds if no new user

## Session Recap Page

Served at `GET /s/<uuid>` from Flask. Single responsive HTML page optimized for mobile.

### Header
- AskGordie logo (dark version, from `assets/`)
- Summary card: "You explored N topics in M minutes" with date/time
- Device location context if available (riding name from device registry)

### Body
- Chat bubbles: user questions (left-aligned, grey background), Gordie answers (right-aligned, red accent `#EC2024`)
- Each Gordie answer has a collapsible "Sources" section with links to canadagpt.ca entity pages
- Timestamps on each exchange

### Footer
- CTA button: "Continue this conversation online" → `canadagpt.ca/register` with `?session=<uuid>&device=<device_id>` params
- Share button: Web Share API on mobile, copy-link fallback on desktop
- Save as PDF button
- Privacy notice: "This session expires in X hours/days. Sign up to save it permanently."
- "Delete this session" link → `POST /s/<uuid>/delete`

### Privacy Banner
Dismissable banner at top: "This conversation was recorded at an AskGordie kiosk. It will be automatically deleted in 24 hours unless you save it."

## Keyboard Prompt Overlay

When a physical keyboard input is detected while in voice view:

1. Body-level `keydown` listener fires on alphanumeric keys
2. Minimal floating input bar appears at the bottom of the screen (not the full prompt view)
3. First keypress character is captured into the input field, cursor focused
4. User types their question and presses Enter
5. Input bar submits via SocketIO `prompt_submit` and dismisses
6. Backend routes through the voice pipeline: LLM → TTS → spoken response
7. Badge continues showing state animations (thinking → speaking)
8. Escape key dismisses the input bar without submitting
9. The kiosk never leaves voice mode — input bar is a temporary overlay

## Supabase Sync

- Background thread on each Pi, runs every 60 seconds
- Queries SQLite for sessions where `synced = 0 AND ended_at IS NOT NULL`
- Posts session + messages to Supabase via httpx using `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY`
- Inserts into `kiosk_sessions` and `kiosk_messages` tables
- On success: sets `synced = 1` locally
- On failure: logs warning, retries next cycle. No data loss.
- Expired unscanned sessions are deleted locally before sync attempt (never sent to cloud)

## Cleanup

Hourly background job on each Pi:

- Delete sessions where `expires_at < now() AND scanned = 0`
- Cascade deletes associated messages (via SQLite FK)
- Log deletion count
- Synced sessions that have expired are also cleaned up locally (cloud retains its copy)

## Privacy (PIPEDA Compliance)

- **Purpose statement**: Visible on kiosk display: "This conversation is temporarily stored. Scan to save it or it will be automatically deleted."
- **Retention**: 24 hours default. Extended to 30 days if QR is scanned. Permanent if user signs up and links session.
- **No PII collected**: QR link is the only identifier. No login required to view.
- **User deletion**: "Delete this session" link on recap page immediately purges from SQLite and marks for deletion in Supabase.
- **Auto-deletion**: Hourly cleanup job enforces expiry.

  sessions/

## Integration Points

- `__main__.py`: Initialize SessionStore, start sync + cleanup threads
- `app.py`: Create session on wake, save messages in pipeline, emit `session_ended` on conversation end
- `persona.py`: New routes (`/s/<uuid>`, `/qr/session/<uuid>`, `/s/<uuid>/delete`), SocketIO handler for session events
- `persona.html`: QR display area in voice view, keyboard overlay input bar
- `persona.js`: QR fade-in/out logic, keyboard listener, session event handlers
